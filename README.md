> A dark, editorial-styled single-page application with a sidebar navigation, dashboard stats, and full CRUD for all study data.


## ✨ Features

- 📊 **Live Dashboard** — See completed sessions, hours studied, task completion rate, and upcoming sessions at a glance
- 📚 **Subject Management** — Create subjects with custom colours; all data is linked to subjects
- 📅 **Session Scheduler** — Plan study blocks with date, start/end time, and notes; mark as completed or skipped
- ✅ **Task Board** — Add tasks with priority levels (High / Medium / Low) and due dates; toggle completion
- 🎯 **Goal Tracker** — Set hour-based study goals per subject with deadline and a live progress bar
- 💾 **Persistent MySQL Storage** — All data is saved to a local MySQL database via XAMPP
- 🔧 **Auto Database Init** — The server automatically creates the database and all tables on first run — no manual SQL needed
- 🌙 **Dark Editorial UI** — A polished, dark-themed interface using DM Serif Display + DM Sans fonts

## 🚀 Getting Started

### ✅ Prerequisites

Make sure you have the following installed:

- [Node.js](https://nodejs.org/) `v18+`
- [XAMPP](https://www.apachefriends.org/) with **MySQL** module
- A modern web browser (Chrome, Firefox, Edge)

---

### 📥 Installation

**1. Clone the repository**
```bash
git clone https://github.com/your-username/study-planner.git
cd study-planner
```

**2. Install dependencies**
```bash
npm install
```

**3. Start XAMPP MySQL**

> Open the **XAMPP Control Panel** and click **Start** next to **MySQL**
> Make sure it is running on port `3306`

**4. Start the server**
```bash
npm start
```

You should see:
```
🔧  Checking database...
✅  Database ready (study_planner)
🚀  Study Planner running at  http://localhost:3000
```

**5. Open the app**

Visit 👉 **[http://localhost:3000](http://localhost:3000)** in your browser

> 🎉 The server automatically creates the `study_planner` database and all tables on first run!

---

### 🔁 Development Mode (auto-restart)

```bash
npm run dev
```

This uses `nodemon` to watch for file changes and restart automatically.

## 🗄️ Database

### Auto-Initialisation (Recommended)

You don't need to do anything! When you run `npm start`, the server will:

1. Connect to XAMPP MySQL on `localhost:3306`
2. Create the `study_planner` database if it doesn't exist
3. Create all 4 tables if they don't exist
4. Your app is ready to use ✅

---

### Manual Setup (Optional)

If you prefer to set up the database manually via **phpMyAdmin**:

1. Open [http://localhost/phpmyadmin](http://localhost/phpmyadmin)
2. In the left panel, click **"New"** and create a database named `study_planner`
3. Select the `study_planner` database (click on it in the left panel)
4. Click the **"Import"** tab at the top
5. Select `setup.sql` and click **Go**

---

### 📐 Database Schema

```
study_planner
├── subjects      (id, name, color, created_at)
├── sessions      (id, subject_id, title, description, date, start_time, end_time, duration_min, status, created_at)
├── tasks         (id, subject_id, session_id, title, due_date, priority, completed, created_at)
└── goals         (id, subject_id, title, target_hours, logged_hours, deadline, created_at)
```

## 🌐 API Reference

All endpoints are served at `http://localhost:3000/api`

### 📚 Subjects
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/subjects` | Get all subjects |
| `POST` | `/api/subjects` | Create a new subject |
| `DELETE` | `/api/subjects/:id` | Delete a subject (cascades to sessions, tasks, goals) |

### 📅 Sessions
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/sessions` | Get all sessions (filter by `subject_id`, `from`, `to`) |
| `POST` | `/api/sessions` | Create a new session |
| `PATCH` | `/api/sessions/:id/status` | Update session status (`planned` / `completed` / `skipped`) |
| `DELETE` | `/api/sessions/:id` | Delete a session |

### ✅ Tasks
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/tasks` | Get all tasks (filter by `subject_id`, `completed`) |
| `POST` | `/api/tasks` | Create a new task |
| `PATCH` | `/api/tasks/:id/toggle` | Toggle task completed status |
| `DELETE` | `/api/tasks/:id` | Delete a task |

### 🎯 Goals
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/goals` | Get all goals |
| `POST` | `/api/goals` | Create a new goal |
| `PATCH` | `/api/goals/:id/log` | Log study hours to a goal |
| `DELETE` | `/api/goals/:id` | Delete a goal |

### 📊 Stats
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/stats` | Get dashboard summary (sessions, hours, tasks, upcoming) |

## ⚙️ Configuration

By default the server connects with XAMPP's default credentials. To change them, edit the pool config in `server.js`:

```js
pool = mysql.createPool({
  host     : "localhost",  // 🔧 Change if MySQL is on a different host
  port     : 3306,         // 🔧 Change if using a different port
  user     : "root",       // 🔧 Change to your MySQL username
  password : "",           // 🔧 Add your MySQL password if set
  database : "study_planner",
});
```
